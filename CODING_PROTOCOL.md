## 2. 実装ルール

### 2.1 コード変更前

- `todo.md` に `[ ]` でタスクを追加してから実装を開始する
- 影響範囲を事前に特定し、変更するファイルをリストアップする
- 関連コンポーネントを `grep` で検索し、依存関係を把握する

### 2.2 コード変更中

- 1つの変更は1つの目的のみ（矢印修正と動画修正を同時に行わない）
- TypeScript エラーは `npx tsc --noEmit` で毎回確認する
- `Pressable` に `className` は使わない（NativeWind の既知問題）
- `FlatList` の `pagingEnabled` と `snapToInterval` を同時に使わない
- `absoluteFillObject` + 高い `zIndex` は他のタッチを阻害するため注意
- モックデータ・ハードコードされた値を本番コードに残さない

### 2.3 コード変更後

- `npx tsc --noEmit` でTypeScriptエラー0を確認
- `todo.md` の該当タスクを `[x]` に更新
- `LATEST.md` を更新（変更内容・チェックポイントIDを記録）
- `webdev_save_checkpoint` でチェックポイントを保存
- `git push github main` でGitHubにプッシュ

## 4. 既知の問題・禁止事項

### 禁止事項

| 禁止 | 理由 |
|------|------|
| `Pressable` に `className` を使う | NativeWind がタッチを無効化する |
| `pagingEnabled` と `snapToInterval` を同時使用 | スナップ計算が競合してスクロール不能になる |
| `absoluteFillObject` + 高 `zIndex` を他コンポーネントの上に重ねる | タッチイベントを横取りする |
| `scrollToIndex` を `getItemLayout` なしで使う | 動作不安定・クラッシュの原因 |
| LLM（Manus組み込み）を本番コードで呼び出す | クレジット消費・コスト発生 |
| モックデータを本番フローに残す | 実データが表示されない |
| `git reset --hard` を使う | `webdev_rollback_checkpoint` を使うこと |

### 既知の制約

- Manus プレビューブラウザは外部画像URL（`i.ytimg.com` 等）をCORSブロックする → `/api/image-proxy` 経由で配信すること
- YouTube RSS フィード（`youtube.com/feeds/videos.xml`）はサンドボックスからアクセス不可 → YouTube Data API v3 を使うこと
- TikTok は iframe 埋め込み不可 → サムネイル表示 + 外部ブラウザで開く方式

## QA 2026-03-18 ローディング遅延改善

### 問題報告
- ユーザー報告：「ロードが長い」「ニュースを読み込んでいます...画面が長く表示される」

### 原因分析
1. サーバーサイド：4つのRSSフィード（NHK・朝日・livedoor・Yahoo）を並列取得していた
2. 各フィードのタイムアウトが6秒に設定されていた
3. クライアント側：refetchInterval が5分と短く、キャッシュ戦略が不十分だった

### 修正内容

#### サーバーサイド（`server/api/japanese-news-client.ts`）
```
- RSSフィードタイムアウト：6秒 → 3秒に短縮
- キャッシュTTL：10分 → 5分に短縮
- 高速パス実装：NHKフィードを最初に取得し、その後他フィードを並列取得
  * 初回ロード時は最速のNHKフィードのみで応答
  * 他フィード（朝日、livedoor、Yahoo）は非同期で追加
```

#### クライアント側（`hooks/use-real-time-data.ts`）
```
- refetchInterval：5分 → 10分に延長（サーバー負荷軽減）
- staleTime：5分 → 10分に延長（キャッシュ有効期間拡大）
- gcTime：新規追加15分（TanStack Query v5のメモリ管理最適化）
```

### 修正後の動作
- ✅ ローディング画面が表示されずに即座にメイン画面が表示
- ✅ 2回目以降のアクセスはキャッシュから高速表示
- ✅ デグレなし（矢印ナビゲーション・記事タップ・外部リンク全て正常）
