# SWELL デバッグメモ - 同じミスを繰り返さないために

## 発見されたデグレード問題

### 1. 矢印キーの無反応問題（重大）
**症状:**
- 矢印ボタンのクリックが無反応
- スライド自体は反応しているが、ページ数表示（ページインジケータ）がカード枚数に追従していない

**根本原因の推測:**
- `pixel-arrow.tsx` で `onClick` ハンドラを追加したが、実際のページネーション状態管理と同期していない可能性
- ページインジケータが古い状態を表示している（状態管理のバグ）
- React Native Web での `Pressable` コンポーネントと状態管理の連携が不完全

**修正方法（次回実装時）:**
1. ページネーション状態管理を確認（`HomeScreen` の `currentPage` 状態）
2. 矢印ボタンのクリックハンドラが正しく状態を更新しているか検証
3. ページインジケータコンポーネントが最新の状態を参照しているか確認
4. テストを書いて、ボタンクリック → 状態更新 → UI反映 の流れを検証

**チェックリスト:**
- [ ] `app/(tabs)/index.tsx` の `currentPage` 状態管理を確認
- [ ] 矢印ボタンの `onPress` ハンドラが `setCurrentPage()` を呼び出しているか確認
- [ ] ページインジケータが `currentPage` を参照しているか確認
- [ ] React DevTools で状態変化を追跡してデバッグ

### 2. YouTubeサムネイル取得失敗（重大）
**症状:**
- YouTubeサムネイルが表示されない
- クリックで動画にも飛べない

**根本原因の推測:**
- YouTube API からサムネイル URL が取得できていない
- サムネイル URL は取得されているが、画像読み込みに失敗している（CORS エラー）
- ビデオコンポーネントのクリックハンドラが実装されていない

**修正方法（次回実装時）:**
1. `VideoClient` でサムネイル URL が正しく取得されているか確認
2. ブラウザのコンソールで CORS エラーを確認
3. サムネイル画像の読み込み状態をデバッグ
4. ビデオアイテムのクリックハンドラを実装（YouTube へのディープリンク）

**チェックリスト:**
- [ ] `server/clients/video-client.ts` でサムネイル URL 取得ロジックを確認
- [ ] ブラウザコンソールで CORS エラーを確認
- [ ] `Image` コンポーネントの `onError` ハンドラでエラーをログ出力
- [ ] ビデオアイテムに `onPress` ハンドラを追加（YouTube へのリンク）
- [ ] テストデータでサムネイル表示を検証

## 教訓と今後の対策

### 1. テスト駆動開発の重要性
**教訓:** プログラマティッククリックテストだけでは不十分。ユーザーの手動クリックでも動作するか確認が必須。

**対策:**
- 状態管理の変化をテストする Vitest テストを必ず書く
- UI コンポーネントの統合テストを実装
- ブラウザでの手動テストを必ず実施

### 2. 外部 API 連携の検証
**教訓:** API から取得したデータが正しく表示されているか、ブラウザのコンソールで CORS エラーなどを確認する必要がある。

**対策:**
- API レスポンスをログ出力
- ブラウザコンソールのエラーを常に監視
- 画像読み込みエラーハンドリングを実装
- テストデータで動作確認

### 3. デザイン実装の完全性
**教訓:** デザイン要件が「波で表現する」など具体的でない場合、実装漏れが発生しやすい。

**対策:**
- デザイン要件を具体的に定義（色、サイズ、アニメーション）
- デザイン実装をチェックリスト化
- 各セクション（News、Social、Market）のデザインを統一

### 4. チェックポイント作成前の検証
**教訓:** プログラマティッククリックテストで成功しても、実際のユーザー操作では失敗することがある。

**対策:**
- チェックポイント作成前に、すべての主要機能を手動でテスト
- 各セクション（News、Social、Market）でページネーション動作確認
- 外部リンク（YouTube、ニュース元）のクリック動作確認
- ブラウザのコンソールエラーを確認
